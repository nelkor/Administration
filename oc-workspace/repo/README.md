# Langley

Langley — frontend-only MVP веб-приложения для тренировки иностранных слов.

Цель проекта: собрать статическое клиентское приложение, в котором пользователь
настраивает тренировку и проходит бесконечную сессию карточек.
Знакомые слова должны появляться реже, незакреплённые — чаще.

## Что должно получиться

Приложение должно позволять:

- выбрать родной язык;
- выбрать изучаемый язык;
- выбрать один или несколько предустановленных наборов слов;
- запустить тренировку;
- переворачивать карточки;
- отмечать знакомство со словом свайпами;
- пользоваться светлой и тёмной темой;
- выйти из тренировки обратно к настройкам.

## Технологическая база

Проект собирается на:

- TypeScript;
- React;
- Vite;
- Zustand;
- Motion;
- CSS Modules;
- CSS custom properties для темизации;
- Vitest.

Подключать дополнительные библиотеки можно только если они дают явную пользу и не размывают архитектуру.

## Источники истины

В проекте используются только три профильных документа:

- `docs/product.md` — продуктовые требования и пользовательское поведение;
- `docs/architecture.md` — архитектурные границы, слои, инварианты и правила модулей;
- `docs/plan.md` — порядок реализации.

## В каком порядке читать документы

Перед разработкой читать документы в таком порядке:

1. `docs/product.md`
2. `docs/architecture.md`
3. `docs/plan.md`

Порядок важен:

- сначала понять, что именно нужно сделать;
- затем понять, как это должно быть организовано;
- затем реализовывать по этапам.

## Что не нужно делать

До выхода MVP не нужно:

- добавлять backend;
- добавлять авторизацию;
- сохранять прогресс между сессиями;
- усложнять алгоритм обучения без необходимости;
- вводить новые архитектурные слои или сущности без явной пользы;
- оставлять мёртвый код, временные решения и неиспользуемые абстракции.

## Как выполнять задачи

Изменение считается завершённым, если:

- реализовано требуемое поведение;
- существующее поведение не сломано;
- решение укладывается в архитектурные границы;
- публичные интерфейсы модулей остаются понятными;
- код проходит проектные проверки;
- решение остаётся локальным, предсказуемым и без лишнего усложнения.
