# План разработки

## Этап 1. Инициализация проекта

### 1.1. Создать проект

- Инициализировать проект на `TypeScript + React + Vite`.
- Подключить базовую структуру директорий.
- Настроить базовые npm-скрипты.
- Убедиться, что проект запускается и собирается.

### 1.2. Подключить базовые зависимости

- Подключить `Motion`.
- Подключить `Zustand`.
- Подключить `Vitest`.
- Добавить только действительно необходимые вспомогательные зависимости, если они нужны сразу.

### 1.3. Подготовить каркас приложения

- Создать корневой `App`.
- Подготовить базовый layout приложения.
- Подготовить точку входа для темизации и глобальных стилей.

Результат этапа:

- проект запускается;
- проект собирается;
- есть рабочий каркас приложения.

---

## Этап 2. Базовая архитектура модулей

### 2.1. Разложить код по модулям

Создать понятные модули для:

- словарных данных;
- настройки тренировки;
- тренировки;
- общих UI-компонентов;
- общих утилит;
- темизации.

### 2.2. Зафиксировать публичные интерфейсы модулей

- В каждом модуле сделать `index.ts`.
- Ограничить внешние импорты только публичными интерфейсами.
- Проследить, чтобы по `index.ts` было понятно, что модуль предоставляет.

Результат этапа:

- структура проекта соответствует архитектуре;
- у каждого модуля есть понятная внешняя граница;
- код готов к дальнейшему наращиванию.

---

## Этап 3. Слой данных словарей

### 3.1. Описать типы данных

- Описать тип словарной записи.
- Описать тип набора слов.
- Описать тип данных, готовых к тренировке.

### 3.2. Подготовить предустановленные данные

- Добавить набор существительных.
- Добавить набор глаголов.
- Поддержать языки `en`, `ru`, `es`.

### 3.3. Подготовить доступ к наборам

- Модуль, который возвращает список доступных наборов слов.
- Модуль, который возвращает пул слов по переданным наборам и языковой паре.
- Поддержать симуляцию загрузки данных.

### 3.4. Реализовать фильтрацию слов

- Фильтровать слова по выбранному родному языку.
- Фильтровать слова по выбранному изучаемому языку.
- Исключать слова, где хотя бы одно нужное значение отсутствует.
- Объединять слова из выбранных наборов в тренировочный пул.

### 3.5. Покрыть слой данных тестами

Проверить:

- корректность структуры данных;
- корректность фильтрации;
- исключение `null`-значений;
- объединение нескольких наборов.

Результат этапа:
приложение умеет получать и готовить корректный набор слов для тренировки.

---

## Этап 4. Модель настройки тренировки

### 4.1. Спроектировать состояние настройки

Добавить состояние для:

- родного языка;
- изучаемого языка;
- выбранных наборов;
- готовности к началу тренировки.

### 4.2. Реализовать правила валидации

- Родной и изучаемый язык не должны совпадать.
- Должен быть выбран хотя бы один набор слов.
- Начало тренировки должно быть возможно только при валидной конфигурации.

### 4.3. Реализовать действия настройки

- Выбор родного языка.
- Выбор изучаемого языка.
- Выбор и снятие наборов слов.
- Подтверждение запуска тренировки.

### 4.4. Покрыть модель настройки тестами

Проверить:

- смену языков;
- запрет одинаковых языков;
- выбор наборов;
- признак готовности к запуску.

Результат этапа:
есть рабочая и предсказуемая модель конфигурации тренировки.

---

## Этап 5. Экран настройки

### 5.1. Реализовать UI выбора языков

- Показать доступные языки.
- Сделать понятный выбор родного языка.
- Сделать понятный выбор изучаемого языка.
- Не допускать состояния, когда языки совпадают.

### 5.2. Реализовать UI выбора наборов

- Показать доступные наборы слов.
- Позволить выбрать один или несколько наборов.
- Отразить текущее состояние выбора.

### 5.3. Реализовать запуск тренировки

- Сделать кнопку запуска.
- Кнопка должна быть недоступна, если конфигурация невалидна.
- При запуске должен "запрашиваться" (формироваться) тренировочный набор.

### 5.4. Обработать состояние загрузки наборов

- Показать загрузку во время симуляции получения данных.
- Обработать успешное получение данных.
- При необходимости показать простое состояние ошибки.

Результат этапа:
пользователь может настроить тренировку и перейти к ней.

---

## Этап 6. Доменная модель тренировки

### 6.1. Спроектировать сущности тренировки

Добавить модели для:

- тренировочного слова;
- состояния слова в текущей сессии;
- очереди / пула слов;
- трёх текущих карточек;
- истории действий пользователя.

### 6.2. Реализовать старт тренировки

- Инициализировать сессию на основе выбранных слов.
- Сформировать первые три карточки.
- Обработать случай малого числа слов.

### 6.3. Реализовать действия над карточкой

- Переворот карточки.
- Отметка "знакомо".
- Отметка "не закреплено".
- Замена ушедшей карточки новой.

### 6.4. Реализовать алгоритм повторного показа

Алгоритм должен обеспечивать:

- более редкое появление знакомых слов;
- более частое появление незакреплённых слов;
- устойчивую бесконечную тренировку.

На этом этапе не нужен сложный алгоритм адаптивного обучения.
Нужен простой, понятный и контролируемый механизм повторного показа.

### 6.5. Реализовать поведение трёх карточек

Пользователь может свайпать любую из трёх видимых карточек.
Когда освобождается место, конструкция "уплотняется" вниз;
на освободившееся место сверху выходит новая карточка.

### 6.6. Реализовать выход из тренировки

- Пользователь должен иметь явный способ выйти из сессии.
- Выход должен возвращать к настройкам.

### 6.7. Покрыть доменную модель тренировки тестами

Проверить:

- инициализацию тренировки;
- формирование трёх карточек;
- переворот карточки;
- обработку левого и правого свайпа;
- повторный показ слов;
- бесконечность тренировки;
- корректный приход следующих карточек.

Результат этапа:
доменная логика тренировки работает независимо от UI.

---

## Этап 7. UI карточек и экрана тренировки

### 7.1. Реализовать экран тренировки

- Создать экран с тремя текущими карточками.
- Сделать читаемую композицию карточек.

### 7.2. Реализовать карточку

- Показ слова на изучаемом языке.
- Переворот карточки.
- Показ перевода на родном языке на обратной стороне.

### 7.3. Реализовать свайпы

- Свайп влево = слово знакомо.
- Свайп вправо = слово не закреплено.
- После свайпа карточка уходит, а лента сдвигается.
- UI должен отражать это быстро и понятно.

### 7.4. Реализовать анимации

- Анимация переворота карточки.
- Анимация ухода карточки.
- Анимация появления новой карточки.
- Допустимы объёмные и выразительные эффекты.

### 7.5. Реализовать вспомогательные элементы экрана

При необходимости:

- подсказки по свайпам;
- кнопка выхода;
- индикаторы текущего состояния;
- небольшие текстовые подсказки.

Результат этапа:
экран тренировки визуально завершён и соответствует основному сценарию продукта.

---

## Этап 8. Темизация

### 8.1. Спроектировать тему

- Подготовить светлую тему.
- Подготовить тёмную тему.
- Вынести цвета и базовые токены в CSS custom properties.

### 8.2. Реализовать переключение темы

- Добавить переключатель темы.
- Обновлять оформление приложения без перезагрузки.

### 8.3. Сохранить выбор

Сохранять тему в `localStorage`.

Результат этапа: приложение поддерживает светлую и тёмную тему.

---

## Этап 9. Полировка UX

### 9.1. Проверить понятность интерфейса

- Убедиться, что настройка понятна без дополнительных объяснений.
- Убедиться, что смысл свайпа читается по интерфейсу.
- Убедиться, что у пользователя нет тупиковых состояний.

### 9.2. Проверить пустые и пограничные случаи

- Нет подходящих слов после фильтрации.
- Выбраны несовместимые настройки.
- Наборы ещё загружаются.
- В наборе очень мало слов.

### 9.3. Проверить адаптивность

- Приложение должно корректно работать на типичном мобильном размере.
- Приложение должно корректно работать на десктопе.
- Карточки и жесты не должны ломаться на узких экранах.

### 9.4. Упростить лишнее

- Удалить шум в интерфейсе.
- Удалить неиспользуемые элементы.
- Упростить спорные визуальные решения, если они мешают взаимодействию.

Результат этапа:
MVP выглядит цельно и не производит впечатления сырого прототипа.

## Минимально необходимый результат MVP

MVP можно считать состоявшимся, если:

- пользователь может выбрать родной язык;
- пользователь может выбрать изучаемый язык;
- пользователь может выбрать один или несколько наборов;
- пользователь может перейти в тренировку;
- на экране тренировки видно три карточки;
- карточки можно переворачивать;
- карточки можно свайпать влево и вправо;
- знакомые слова действительно появляются реже;
- незакреплённые слова действительно появляются чаще;
- тренировка не заканчивается сама;
- приложение поддерживает светлую и тёмную тему;
- проект запускается, работает и проходит проверки.
