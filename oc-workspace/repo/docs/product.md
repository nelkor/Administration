# Product

## Назначение

Langley — веб-приложение для тренировки иностранных слов.

Пользователь выбирает языковую пару и один или несколько наборов слов,
после чего запускает бесконечную тренировку карточек. Слова, которые
пользователь знает лучше, должны появляться реже. Слова, которые пользователь
знает хуже, должны появляться чаще.

## Пользовательская ценность

Приложение должно:

- быстро приводить пользователя к тренировке без лишних шагов;
- давать понятную обратную связь по каждой карточке;
- удерживать непрерывный поток слов без ручного перезапуска сессии;
- адаптировать частоту повторов под ответы пользователя;
- оставаться простым и предсказуемым в использовании.

## Пользовательский сценарий

### 1. Вход в приложение

При открытии приложения пользователь попадает на экран настройки тренировки.

На этом экране он должен иметь возможность:

- выбрать родной язык;
- выбрать изучаемый язык;
- выбрать один или несколько доступных наборов слов;
- запустить тренировку, если выбранные параметры валидны.

### 2. Запуск тренировки

После запуска тренировки приложение формирует рабочую выборку из доступных слов выбранных наборов.

В тренировку попадают только слова, которые:

- входят хотя бы в один выбранный набор;
- имеют значение на изучаемом языке;
- имеют значение на родном языке.

### 3. Прохождение тренировки

Во время тренировки пользователь видит экран карточек.

Ожидаемое поведение:

- на экране одновременно присутствуют три текущие карточки;
- любую карточку можно перевернуть, чтобы увидеть перевод;
- после свайпа карточка покидает экран;
- на её место подаётся карточка, которая была выше неё;
- структура уплотняется вниз, сверху приходит одна новая карточка.

### 4. Оценка знания слова

Пользователь оценивает слово свайпом:

- свайп влево — слово знакомо хорошо;
- свайп вправо — слово знакомо недостаточно хорошо.

Эта оценка должна влиять на вероятность
повторного появления слова в текущей тренировке:

- хорошо знакомые слова должны возвращаться реже;
- плохо закреплённые слова должны возвращаться чаще.

Точный алгоритм может быть простым,
но наблюдаемое поведение должно соответствовать этим правилам.

### 5. Выход из тренировки

Пользователь может в любой момент выйти из тренировки
и вернуться к экрану настройки.

## Экран настройки

Экран настройки должен отвечать на три вопроса:

- на каком языке пользователь уже понимает значение;
- какой язык он тренирует;
- из каких наборов брать слова.

Требования к экрану:

- выбранные параметры должны быть визуально понятны;
- недопустимая конфигурация не должна запускать тренировку;
- пользователь должен понимать, что нужно выбрать до старта;
- если данные наборов подгружаются с задержкой, интерфейс должен это явно показывать.

## Экран тренировки

Экран тренировки должен быть сосредоточен на одной основной задаче — быстрой оценке карточек.

Требования к экрану:

- главный акцент должен быть на активной карточке;
- переворот карточки должен явно разделять слово и перевод;
- направление свайпа должно быть однозначно связано со смыслом ответа;
- после свайпа интерфейс должен быстро выводить следующую карточку;
- пользователь должен иметь явный способ покинуть тренировку.

## Темы оформления

Приложение поддерживает светлую и тёмную тему.

- обе темы должны быть доступны из пользовательского интерфейса;
- переключение темы не должно ломать читаемость карточек и элементов управления;
- тема должна применяться ко всему приложению согласованно.

## Поддерживаемые языки

Поддерживаются языки:

- `en`;
- `ru`;
- `es`.

Пользователь может выбирать родной и изучаемый язык из этого набора.

## Наборы слов

Доступны два предустановленных набора:

- 100 наиболее употребимых существительных;
- 100 наиболее употребимых глаголов.

Пользователь может выбрать любой один набор или оба набора одновременно.

## Требования к данным слов

Каждое слово, используемое в тренировке,
должно быть пригодно для показа в выбранной языковой паре.

## Правила поведения тренировки

Поведение тренировки должно быть устойчивым и предсказуемым:

- сессия бесконечна и завершается только явным действием пользователя;
- после каждой оценки тренировка должна оставаться в валидном состоянии;
- слова с разной оценкой должны постепенно расходиться по частоте повторов;
- интерфейс не должен требовать от пользователя дополнительных подтверждений после каждого свайпа.

## Пограничные случаи

- пользователь ещё не выбрал нужные параметры для запуска;
- выбранные наборы загружаются или имитируют загрузку;
- после фильтрации не осталось ни одного слова для выбранной пары языков;
- пользователь выходит из тренировки и снова запускает её с другой конфигурацией.

В каждом из этих случаев приложение должно сохранять
понятное и управляемое поведение без ошибок и тупиковых состояний.

## Критерий готовности продукта

MVP можно считать реализованным, если пользователь может:

1. открыть приложение;
2. выбрать родной язык;
3. выбрать изучаемый язык;
4. выбрать один или несколько наборов;
5. запустить тренировку;
6. переворачивать карточки;
7. оценивать слова свайпами;
8. наблюдать, что знакомые слова появляются реже, а незнакомые — чаще;
9. переключать тему;
10. выйти из тренировки и начать новую.
