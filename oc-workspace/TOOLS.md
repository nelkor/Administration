## Рабочие пути

### ~/.openclaw/workspace/state.json

В этом файле хранится текущее состояние работы:
ведётся ли работа и что надо делать.

Интерфейс:

- **stopWorking** boolean (если true — брать задачи запрещено)
- **task** 'implement' | 'review' | 'reconsider' | 'merge' | 'fix' | 'audit'
- **target** null | number (номер Issue или Pull Request, если нужен задаче)

Пример:

```json
{
  "stopWorking": false,
  "task": "audit",
  "target": null
}
```

### ~/.openclaw/workspace/repo/

Продукт, над которым ведётся разработка. Основная ветка — `main`.
Состояние продукта определяется файлами:

- **README.md** суть продукта
- **docs/** спецификация, архитектура, план работ и так далее

### ~/.openclaw/workspace/tasks/

- implement.md
- review.md
- reconsider.md
- merge.md
- fix.md
- audit.md

Инструкции по видам задач.

## Инструкция по разработке

Ты занимаешься разработкой только по пробуждению.
При общении в чате ты не можешь самовольно переходить к разработке.

### Флаг блокировки

Перед тем, как выполнить задачу, проверь файл `state.json`.
Если `stopWorking: true`, заниматься разработкой запрещено,
даже если тебя просят.

Если ты проверил `state.json` и `stopWorking: false`,
то бери задачу и сразу установи `stopWorking` в `true`.
Это требуется для того, чтобы никто не смог взять задачу параллельно.

### Виды задач

- **implement** написать код или прийти к выводу, что это невозможно
- **review** одобрить задачу или прийти к выводу, что она нуждается в доработке
- **reconsider** переработать Issue
- **merge** слить Pull Request в основную ветку
- **fix** то же, что `implement`, но доработка существующего Pull Request
- **audit** решить, чем заниматься дальше

Прочитай соответствующий файл в директории `tasks` и только после этого
приступай к задаче. Если такого файла нет — остановись.

Если `state.json` или сопутствующие данные некорректны
(например, Issue с номером `target` не существует) — остановись.

### После выполнения

Разработка ведётся пошагово. Одна задача — один шаг.
После выполнения шага необходимо подготовить следующий.

Заполни `state.json`: следующие `task` + `target`,
установи `stopWorking` в `false`.

Нельзя выполнять более одного шага за раз. Если подготовил следующий шаг
— остановись. Чтобы ты не забыл это правило, я добавил пометку в начало
каждого файла в `tasks`:

> **ВАЖНО**
> Подготовка следующего шага — это только редактирование `state.json`.
> После подготовки следующего шага продолжать работу нельзя!
