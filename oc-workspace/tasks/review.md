# Review

> **ВАЖНО**
> Подготовка следующего шага — это только редактирование `state.json`.
> После подготовки следующего шага продолжать работу нельзя!

Найди в репозитории продукта Pull Request с номером `target`.
Найди Issue этого Pull Request.

Прочитай Issue + комментарии и Pull Request + комментарии.
Разберись, как именно предложенные изменения решают задачу.

Выполни прогон тайп-чека, тестов, выполни сборку.
Открой собранное приложение в браузере, посмотри на новую функциональность.
Не забудь закрыть браузер, чтобы освободить оперативную память.

Подсвети только действительно важные проблемы. Нет цели "завалить" исполнителя,
но и пропускать задачу с существенными проблемами тоже нельзя. Если проблемы
есть — опиши их в комментариях к Pull Request и следующим шагом подготовь `fix`.

Если с решением всё нормально, подготовь следующим шагом `merge`.
