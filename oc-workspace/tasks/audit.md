# Audit

Определи текущее состояние продукта.
Чтобы узнать, какой сейчас этап плана, посмотри последние Issues.

Соответствуют ли модули исходного кода документации?

Собери приложение и запусти его в браузере.
Посмотри, как работает заявленная функциональность.
Не забудь закрыть браузер, чтобы освободить оперативную память.

Если есть существенные проблемы, сформулируй задачу на их исправление.
Если таких проблем нет, сформулируй следующую задачу по плану разработки.

Если план выполнен — остановись.

Оформи сформулированную задачу в Issue в GitHub.
Создай ветку. Прикрепи созданную ветку к Issue.

В качестве следующего шага подготовь `implement`
с номером созданного Issue в качестве `target`.
