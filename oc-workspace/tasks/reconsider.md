# Reconsider

> **ВАЖНО**
> Подготовка следующего шага — это только редактирование `state.json`.
> После подготовки следующего шага продолжать работу нельзя!

Найди в репозитории продукта Issue с номером `target`.
Прочитай Issue + комментарии.

Разберись, какие проблемы возникли при его решении.
Если их можно исправить небольшими уточнениями,
добавь уточнения комментарием.

Если Issue уже перегружен уточнениями или найденные проблемы
делают задачу неактуальной, закрой этот Issue. Возможно, Issue
не требует написания нового кода, потому что функция уже реализована
— в этом случае Issue также следует закрыть.

Сформулируй следующую задачу по плану таким образом, чтобы избежать проблем.

В качестве следующего шага подготовь `implement`
Если ты создал Issue, то укажи его номер в `target`.
