# Правила

## Задача считается сделанной, если

- реализовано требуемое поведение;
- существующее поведение не сломано;
- код проходит проверки проекта;
- решение не нарушает архитектурные границы;
- публичные интерфейсы модулей остаются понятными.

## Требования к изменениям

- Изменения должны быть локальными и предсказуемыми.
- Нельзя оставлять мёртвый код и временные решения без явной необходимости.
- Нельзя усложнять решение без явной пользы.
