# Правила

## Общие правила

- Код должен быть простым, читаемым и предсказуемым.
- Не допускаются лишние абстракции и преждевременное усложнение.
- UI, доменная логика и данные должны быть разделены.
- Каждый модуль должен иметь понятную и устойчивую внешнюю границу.

## Правила модулей

- Каждый модуль должен экспортировать свой публичный интерфейс через `index.ts`.
- По `index.ts` должно быть понятно, что модуль предоставляет наружу.
- Внешний код должен импортировать модуль через его `index.ts`, а не через внутренние файлы.
- Внутреннее устройство модуля можно менять, не ломая его публичный интерфейс без необходимости.

## Правила изменений

- Новая функциональность не должна ломать существующее поведение.
- Логику нельзя прятать в UI-компоненты без необходимости.
- Алгоритмы и вспомогательные функции должны быть вынесены из визуального слоя.
- Каждое изменение должно оставлять проект понятным для следующего прохода ИИ-агента.

## Правила зависимостей

- Новые зависимости допустимы только при явной пользе.
- Нельзя добавлять зависимость, если задачу разумно решает текущий стек.
- Предпочтение отдаётся хорошо документированным библиотекам.
